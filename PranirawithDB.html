Complete Hosting & Database Guide

Overview

This website uses IndexedDB (browser-based database) for data storage. No external database server required! However, for production hosting, we recommend setting up a proper backend.

Option 1: Static Hosting (Current - No Database Server Needed)

How it works:

  • Data stores in browser's IndexedDB (local storage)
  • Each device has its own separate database
  • Data persists until browser cache is cleared
  • Perfect for single-admin usage

Steps to Host:

  1. Save this HTML file as index.html
  2. Upload to any static hosting: GitHub Pages, Netlify, Vercel, or GoDaddy/Hostinger
  3. For GoDaddy: Upload to public_html folder via File Manager
  4. For Hostinger: Use hPanel File Manager or FTP

Option 2: Full Database Setup (Recommended for Multi-user)

Recommended Stack:

PHP + MySQL (Works on both GoDaddy & Hostinger)

Step-by-Step Implementation:

1. Database Schema (MySQL)
-- Create Database
CREATE DATABASE pranira_industries;
USE pranira_industries;

-- Products Table
CREATE TABLE products (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    category VARCHAR(100),
    price VARCHAR(100),
    description TEXT,
    specifications TEXT,
    images JSON,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Enquiries Table
CREATE TABLE enquiries (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255),
    company VARCHAR(255),
    email VARCHAR(255),
    phone VARCHAR(20),
    product VARCHAR(100),
    message TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Tickets Table
CREATE TABLE tickets (
    id INT AUTO_INCREMENT PRIMARY KEY,
    ticket_number VARCHAR(20) UNIQUE,
    customer_name VARCHAR(255),
    phone VARCHAR(20),
    equipment_model VARCHAR(100),
    serial_number VARCHAR(100),
    type ENUM('service', 'complaint'),
    issue_description TEXT,
    preferred_date DATE,
    status ENUM('pending', 'in-progress', 'resolved') DEFAULT 'pending',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Admin Table
CREATE TABLE admin (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) UNIQUE,
    password VARCHAR(255),
    logo_path VARCHAR(255)
);
2. Database Connection (PHP)
<?php
// config.php
$host = 'localhost';
$dbname = 'pranira_industries';
$username = 'your_db_username';
$password = 'your_db_password';

try {
    $pdo = new PDO("mysql:host=$host;dbname=$dbname", $username, $password);
    $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
} catch(PDOException $e) {
    die("Connection failed: " . $e->getMessage());
}
?>
3. GoDaddy Hosting Setup
  • Login to GoDaddy cPanel
  • Go to MySQL Database Wizard
  • Create database: pranira_industries
  • Create user and assign all privileges
  • Use phpMyAdmin to import the SQL schema above
  • Upload PHP files to public_html using File Manager or FTP
4. Hostinger Setup
  • Login to hPanel
  • Go to Database → MySQL Databases
  • Create new database and user
  • Open phpMyAdmin from dashboard
  • Import the SQL schema
  • Use File Manager or FTP to upload files to public_html
5. API Endpoints (PHP Examples)
// api/products.php - Get all products
<?php
header('Content-Type: application/json');
require_once 'config.php';

$stmt = $pdo->query("SELECT * FROM products ORDER BY id DESC");
$products = $stmt->fetchAll(PDO::FETCH_ASSOC);
echo json_encode($products);
?>

// api/add_product.php - Add product
<?php
header('Content-Type: application/json');
require_once 'config.php';

$data = json_decode(file_get_contents('php://input'), true);
$stmt = $pdo->prepare("INSERT INTO products (name, category, price, description, specifications, images) VALUES (?, ?, ?, ?, ?, ?)");
$stmt->execute([$data['name'], $data['category'], $data['price'], $data['description'], $data['specifications'], json_encode($data['images'])]);
echo json_encode(['success' => true, 'id' => $pdo->lastInsertId()]);
?>

Running the Program

  1. Local Testing: Simply open the HTML file in any modern browser
  2. With Database: Install XAMPP/WAMP, place files in htdocs, start Apache & MySQL
  3. Production: Upload to hosting provider's public_html folder
  4. Admin Access: Default login - Username: admin, Password: admin

Important Notes

  • Change default admin password immediately after first login
  • For production, implement proper password hashing (bcrypt)
  • Enable HTTPS/SSL certificate on your domain
  • Regular backup of database recommended
  • Current version uses client-side storage (IndexedDB) - data is browser-specific
